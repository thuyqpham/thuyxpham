Lynx Fleet Web and Mobile

Lead Product Design
project brief

A desktop web app for the Phase 3 of the Fair Fares program to be launched Feb 2020. The 3.0 system is required to scale to support about 500K clients up from 50K clients in current version.

This is a heavy data app with each page having many different states of variation dependent on the data input. The app itself is used to approve & manage applications for the FF program. Applications come from mobile app managed by another agency. Once applications are approved clients are sent half priced Metrocards. The FF app is also used to manage issuance and reporting of Metrocards with automated directives and tracking thru 3rd party mail vendor.

The app uses popular front-end frameworks; Angular Material, Bootstrap & Kendo UI, therefore a design system / pattern library had to be created that is amalgation of all 3 frameworks so that visual consistency was maintained across the app.

The project adopts agile methodology with outside agency submitting user stories through Jira. At each point in time there were 3 iteration of the app that require separate protoype to be managed; dev version, QA version & future proposal version.

the challenge of iteration

The Fair Fares web app has already in production for 2 years and it was a challenge to iterate on design due to complexity of app.

As more features were added to the app, UI became cluttered and diminished user friendliness. At the same time, due to scope and timeframe of project roadmap, I had to minimize customization to front-end framework to avoid re-work of prior development. Project objectives:

  • Create clean usable UI, reducing clutter whenever possible
  • Create design system by synthesizing 3 popular front-end frameworks already in use by dev team
  • Create design roadmap for improvements to coincide with product roadmap for development
